──────────────────────────────────────────────────────────
PALABRAS
──────────────────────────────────────────────────────────

explain         ( -- )
  Narra en lenguaje natural lo que hizo la última operación
  AI: qué datos se procesaron, qué resultó, métricas clave.
  Ejemplo:
    s" iris.csv" data-load   \ carga datos
    explain                  \ "Se cargó iris.csv con 150 filas..."

ai-why          ( -- )
  Interpreta los resultados de la última operación: por qué
  el score es alto/bajo, si hay riesgo de sobreajuste, si los
  clusters son buenos, etc. Añade contexto cualitativo.
  Ejemplo:
    rf-model model-train
    ai-why    \ "Score 0.98 en train — posible sobreajuste..."

ai-next         ( -- )
  Sugiere el siguiente paso lógico según la operación anterior
  y el estado actual (¿hay target? ¿hay split? ¿hay modelo?).
  Da comandos Forth concretos y listos para copiar.
  Ejemplo:
    data-split
    ai-next   \ "rf-model model-train   \ Random Forest..."

ai-verbose      ( -- )
  Activa el modo detallado. A partir de ese momento, cada
  operación AI imprime una explicación breve automáticamente.
  Equivale a tener 'explain' después de cada comando AI.

ai-quiet        ( -- )
  Desactiva el modo detallado. Salida mínima (por defecto).

──────────────────────────────────────────────────────────
FLUJO TÍPICO
──────────────────────────────────────────────────────────

\ Opción A: preguntas manuales
  s" datos.csv" data-load
  explain           \ ¿qué ocurrió?
  ai-why            \ ¿por qué esos valores?
  ai-next           \ ¿qué hago ahora?

\ Opción B: modo automático
  ai-verbose
  s" datos.csv" data-load
  data-info
  s" precio" data-target
  data-split
  rf-model model-train
  model-eval
  ai-quiet          \ vuelve al modo silencioso

\ Las tres preguntas se pueden encadenar:
  rf-model model-train
  explain ai-why ai-next
